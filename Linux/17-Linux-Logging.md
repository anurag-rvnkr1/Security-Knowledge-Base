# Introduction

Every action performed on a Linux system leaves traces.

These traces are called **logs**.

Logs record:

- User logins
- System startup
- Service activity
- Hardware events
- Application errors
- Security events
- Network events
- Authentication attempts

Linux logging is one of the most important skills for:

- Linux Administration
- System Engineering
- DevOps
- Site Reliability Engineering (SRE)
- Cybersecurity
- Digital Forensics
- Incident Response
- Threat Hunting

Without logs, troubleshooting and security investigations become extremely difficult.

# What is Logging?

Logging is the process of recording events generated by:

- Kernel
- Operating system
- Applications
- Services
- Security software
- Network components
- Users

Logs provide historical records that help administrators understand what happened on a system.


# What is `logrotate`?

`logrotate` automatically:

- Rotates logs
- Compresses old logs
- Removes expired logs
- Creates new log files
- Preserves configurable retention periods

---

# Log Rotation Workflow

```text
Log File

↓

Rotate

↓

Compress

↓

Archive

↓

Delete Old Logs
```

---

# Configuration Directory

Most systems store configuration in:

```text
/etc/logrotate.conf
```

Additional rules:

```text
/etc/logrotate.d/
```

---

# Example Configuration

```text
/var/log/application.log {

    weekly

    rotate 4

    compress

    missingok

    notifempty

}
```

Meaning:

| Directive | Description |
|------------|-------------|
| weekly | Rotate every week |
| rotate 4 | Keep four archives |
| compress | Compress old logs |
| missingok | Ignore missing files |
| notifempty | Do not rotate empty logs |

---

# Common Directives

| Directive | Purpose |
|------------|----------|
| daily | Rotate daily |
| weekly | Rotate weekly |
| monthly | Rotate monthly |
| rotate N | Keep N archived logs |
| compress | Compress archives |
| delaycompress | Delay compression by one rotation |
| create | Create a new log after rotation |
| copytruncate | Copy then truncate the original log |
| maxsize | Rotate when log exceeds a size |
| size | Rotate based on file size |

---

# Why `copytruncate`?

Some applications keep log files open.

`copytruncate`:

```text
Copy Log

↓

Create Archive

↓

Truncate Original

↓

Application Continues Writing
```

Whenever possible, applications that support reopening log files via signals or built-in mechanisms are generally preferred over `copytruncate`, as truncation has trade-offs.

---

# Testing `logrotate`

Dry run:


Force rotation:

```bash
logrotate -f /etc/logrotate.conf
```
