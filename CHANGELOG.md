# Versioning Strategy

This repository follows **Semantic Versioning**.

### Major Releases (X.0.0)

Breaking organizational or structural changes.

Examples:

* Major repository redesign.
* Large documentation restructuring.
* Significant content reorganization.

---

### Minor Releases (1.X.0)

New educational content.

Examples:

* New cybersecurity topics.
* New labs.
* Additional diagrams.
* New detection engineering content.
* Expanded cloud security notes.

---

### Patch Releases (1.0.X)

Maintenance updates.

Examples:

* Grammar improvements.
* Typo fixes.
* Broken links.
* Markdown formatting.
* Image corrections.
* Minor documentation enhancements.
