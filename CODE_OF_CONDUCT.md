# Code of Conduct

## Our Commitment

We are committed to fostering an open, welcoming, inclusive, and respectful environment for everyone who participates in this project.

This repository exists to share cybersecurity knowledge, encourage responsible learning, and support collaboration within the security community. Every contributor, regardless of experience level, background, or identity, deserves to be treated with professionalism and respect.

---

# Our Values

As members of this community, we strive to:

* Be respectful and professional.
* Share knowledge openly and responsibly.
* Encourage curiosity and continuous learning.
* Accept constructive feedback gracefully.
* Focus discussions on technical accuracy and educational value.
* Support newcomers while maintaining high-quality standards.
* Promote ethical and defensive cybersecurity practices.

---

# Expected Behavior

Examples of behavior that contribute to a positive environment include:

* Treating others with courtesy and respect.
* Being open to different viewpoints and experiences.
* Giving constructive and actionable feedback.
* Helping improve documentation and educational content.
* Reporting issues responsibly.
* Respecting project maintainers' decisions.
* Using inclusive and professional language.

---

# Unacceptable Behavior

The following behaviors are not acceptable:

* Harassment, intimidation, or discrimination of any kind.
* Personal attacks, insults, or abusive language.
* Trolling or intentionally disruptive behavior.
* Publishing private or confidential information without permission.
* Spam, advertising, or unrelated promotional content.
* Encouraging or facilitating unauthorized access to systems.
* Sharing malicious code intended for illegal or unethical use.
* Any activity that violates applicable laws or responsible disclosure practices.

---

# Responsible Security Discussions

This repository is intended for educational, defensive, and authorized security research.

Contributors should:

* Focus on defensive techniques and best practices.
* Encourage responsible disclosure.
* Avoid publishing content that enables unauthorized attacks.
* Clearly distinguish between educational demonstrations and real-world misuse.

---

# Reporting Concerns

If you encounter behavior that violates this Code of Conduct, please open a private communication with the project maintainer.

When reporting an issue, include:

* A description of the incident.
* Relevant context or links.
* Any supporting evidence, if applicable.

All reports will be reviewed fairly and handled with discretion.

---

# Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

Depending on the severity of a violation, maintainers may:

* Request changes to comments or contributions.
* Remove inappropriate content.
* Temporarily restrict participation.
* Permanently remove contributors from the project.

Enforcement decisions will be made in good faith with the goal of maintaining a healthy and productive community.

---

# Scope

This Code of Conduct applies to:

* GitHub Issues
* Pull Requests
* Discussions
* Documentation contributions
* Project-related communications
* Any community spaces associated with this repository
